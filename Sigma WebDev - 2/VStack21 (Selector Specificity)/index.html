Cascade Algorithm

  1. If the type of selectors is same then the selector having the most recent (i.e. present at the bottomost) occurence will win.
  2. If there are different types of selectors then the order of specificity is:
    Inline Style > ID Selector > Class or Attribute Selector > Element Selector > Universal Selector
    (Since, the selectors at the left side are more specific than those present on the right side...thus the more specific a selector is...then there would be more chances of it winning)
    [PRO TIP:Try to use the least specific selectors at first so that it is easy to overrite them later on]
  3. Origin: In case there is an element in which the user hadn't applied css...then in such a case some default css will be applied from the browser's side. Browser has its own stylesheet known as 'user-agent stylsheet'. And this has the least priority since the user defined styles can easily overrite them in case css is applied by the user themselves.
  4. The importance of important flag: This special flag can overrite anything even inline css. So this has the power to change the priority of any selector. Refer to the given link in order to know more about each of the 4 laws usage will examples.

Specificity Calculation

To calculate specificity, assign a value to each part of the selector:
  • Universal Selector: 0
  • Element selectors and pseudo-elements: 1
  • Class selectors, attribute selectors, and pseudo-classes: 10
  • ID selectors: 100
  • Inline styles: 1000
  • !important adds 10000 points to Specificity

    • Remember: More specificity = more power (specificity tells how much a specific a element has been made using the selectors use)

For complex selectors (which have multiple selectors in a single line at one while applying properties) we would just add the values of individual specificities to know the overall specificity

BEST PRACTICES IN CSS

The best practice is keep your webpage code simple and don't target the same elements with multiple selectors. Try to avoid being very specific and if you are being specific (like using id selector or something) then be sure that you won't be overriting it again in future. Otherwise, its best to use just classes as they can be re-used again and again. Try to avoid inline css or important keywords as no-one wants to be too deep or specific while selecting elements so just keep your page simple.
