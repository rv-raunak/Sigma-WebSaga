Media Queries are useful in making responsive webistes. As we can chnage styles according to the device we are operating on. Earlier, when CSS 2 was introduced then it was called as media rule (which had the purpose that we can change style if the user logins from a different device). And in CSS 3 this media rule was changed to media query.

I am a Aqua body guy

While designing repsonsive websites we have the concept of break-points that we need to understand. We use break our webistes into 2-3 breakpoints. We can take some arbitary numbers like 455px or you make any other good approximation. let's say we consider devices less than 450px as phones and from 450px tp 750px we consider them as tablet then any devices further than that like those upto 1080px...then we also have upto 1920 or upto the any number of pixels that we have on our widescreen...and then we can also have ultra-wide moniters having very wide screens. So we can make our website responsive for each of the devices using media queries by changing styles for each devices. (All the numbers taken here are approximation so confirm them once)

Also, we would use media queries only while using raw css. But if you are using frameworks, then we would not explicitly use them since frameworks like bootstrap and tailwind css already use media-queries behind the hood.
